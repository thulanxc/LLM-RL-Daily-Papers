今日主题：验证从「事后裁判」走向「过程共演」。 本期 17 篇新论文呈现三条交织的主线： (1) 验证 / 奖励建模的结构化重构——AgentV-RL（2604.16004）将奖励模型变成一个具备前向/后向双智能体、调用工具、多轮推理的「审计系统」，4B 即超越最强 ORM 25.2%；QuantumQA（2604.18176）把物理求解器的确定性信号与语义评审做自适应融合，让 8B 模型在量子推理上媲美闭源大模型；AtManRL（2604.16158）通过可微注意力显著性直接把「CoT 是否真正参与预测」量化为可优化的奖励项。

(2) 智能体训练从静态走向「环境—数据—策略」共演化——CoEvolve（2604.15840）用 rollout 失败模式反向驱动任务合成，在 Qwen3-30B-A3B 上取得 +18.14% 绝对提升；AutoSearch（2604.17337）提出「最小充分搜索深度」作为奖励，在保持精度的同时大幅压缩 agentic RAG 的冗余搜索；ClawEnvKit（2604.18543）把环境生成本身自动化——1,040 个训练环境以人工方案 1/13,800 的成本覆盖 24 类任务；MindDR 同期方向中 π-Play（2604.14054）给出了无外部数据的多智能体自对弈路线。

(3) 探索与对齐的理论回潮——Demystifying Online Alignment（2604.17207）证明贪心在线 RLHF / DPO 在「决策中心」遗憾度量下达到常数 O(1) 遗憾，为经验上的「超预期效果」给出第一份正式解释；Entropy Control Analysis（2604.09676）则把现有熵控策略（entropy bonus、clipped entropy、KL-reg、自适应调度）统一在一张收敛—探索界定的图上；KnowRL（2604.12627）把「提示即最小充分集」转化为约束子集搜索，缓解硬题零梯度坍塌。

另外值得关注：HorizonBench（2604.17283）把个性化对齐问题推到 163K-token 长上下文、6 个月对话史的「信念更新失败」诊断；LAnR（2604.17866）完全抛弃显式检索查询，在 LLM 潜空间内一气呵成编码/检索/生成；Semantic Equivalence Self-Play（2604.17010）把 Liquid Haskell 形式化验证当作 RL 环境的验证器；LLM Graph Priors for MARL（2604.17191）证明 1.5B 模型已足以为多智能体 RL 生成有效的协调图先验；MT-GRPO（2604.02869）则是第一套落到真实客服场景的多轮工具调用 GRPO 训练栈。

一句话：奖励、验证、探索、数据四件事，正在同时从「外挂的子模块」被吸收进「模型与环境的共演化轨道」。

一、奖励建模与验证 · Reward Modeling & Verification

AgentV-RL: Scaling Reward Modeling with Agentic Verifier

作者团队：Agentic Verifier 研究组（完整名单见 arXiv 原文）
arXiv: 2604.16004 · Reward ModelingAgenticTool-Use
核心贡献：将奖励模型从「单轮打分器」改造为多轮、工具增强的「审计系统」。Agentic Verifier 由两个互补的智能体组成——Forward Agent 从前提逐步推到结论；Backward Agent 反过来用结论去核对每一条前提。两者在多轮内交替调用工具并做内部推理。 训练管线：一个端到端的合成数据引擎自动生成「验证轨迹」；两阶段训练：先 rejection sampling SFT，再做 RL（探索式交互训练）。 结果：4B 规模的 Agentic Verifier 在多个基准上超越 SOTA ORM 25.2%，证明「把 verifier 做成 agent」比「做大单步打分器」更有效。
方法要点：Forward/Backward Agent 分工 + 工具使用（计算器、Python、检索等） + RL 自主决定何时调用何种工具验证。
启示：
  • 本期范式转折点之一：奖励模型不再是冻结的判别器，而是一个「会思考、会用工具、会反查」的智能体。
  • 4B 超 SOTA ORM，说明「结构」比「参数量」更关键——这和 AgentV-RL 前后呼应的一批 Generative Critic 工作（GenAC, 2604.10701）方向一致。
  • 对 Pipeline 影响：RLHF 中 RM 也许需要被重写为 RM-Agent，带来的是 verifier rollout 成本上升，但 reward 质量更稳。

AtManRL: Towards Faithful Reasoning via Differentiable Attention Saliency

Max Henning Höth, Kristian Kersting, Björn Deiseroth, Letitia Parcalabescu
arXiv: 2604.16158 · Reasoning RLGRPOFaithful CoT
问题：CoT 是否真的参与了最终答案的产生？还是仅仅是「事后合理化」？AtManRL 用可微注意力遮罩（learnable additive attention mask）直接量化——如果屏蔽掉 CoT 中某 token 不会损害答案，则该 token 是「无关装饰」；损害越大，则该 token 越「因果重要」。 方法：将这个 mask 扰动下的答案准确率下降，作为 token 的 saliency 奖励，与结果奖励加权合成，在 GRPO 框架里联合优化。这样模型被直接激励生成「真实用到」的 CoT。 实验：在 Llama-3.2-3B-Instruct 上，于 GSM8K 与 MMLU 验证 faithful CoT 与准确率可同时提升。
方法要点：Differentiable Attention Mask + Saliency Reward + GRPO 结果奖励融合。核心公式：r_total = r_outcome + λ · Δaccuracy(mask)。
启示：
  • 「Latent CoT」位置论文（昨日 2604.15726）警告 surface CoT 不可信；AtManRL 给出了一条用可微显著性把「表面 CoT」拉回「因果 CoT」的具体算法路径。
  • 对 PRM 的补充：PRM 打的是步骤正确性；AtManRL 打的是步骤是否真的被使用——正交且互补。

On the Rejection Criterion for Proxy-Based Test-Time Alignment

arXiv: 2604.16146 · RLHFTest-TimeAlignment
核心贡献：分析代理奖励模型（proxy RM）指导的推理期对齐（rejection sampling / best-of-N）中，「什么时候拒绝重采」这一关键决策。作者给出一个在对齐收益与计算开销间取得最优折衷的原则化拒绝准则，并在理论上分析其 alignment-efficiency Pareto 前沿。 结果：相比固定阈值的 BoN，按此准则动态调整可以在相同算力下拿到更好的对齐质量，或在相同对齐水平下显著减小采样次数。
启示：与本月早先 Flexible Empowerment BoN（2604.15614，已收录）系列一脉相承——BoN 正在从「工程 trick」走向「可分析的推理期 RL 算子」。

二、推理 RL 与可验证奖励 · Reasoning RL & RLVR

QuantumQA: Enhancing Scientific Reasoning via Physics-Consistent Dataset and Verification-Aware RL

arXiv: 2604.18176 · RLVRScientific ReasoningVerifiable Rewards
问题设定：LLM 在一般推理上很强，但在需要严格物理约束的量子力学等领域可靠性不足。论文提出 QuantumQA：一个任务自适应构建的大规模数据集，并配以混合验证协议——确定性求解器（物理模拟 / 精确解算）+ 语义审查器双通道。 训练方法：Verification-aware Reward Model（VRM）使用自适应奖励融合机制，动态整合来自科学执行套件的确定性信号和多维语义评估。RLVR 在此融合奖励下进行策略训练。 结果：8B 优化后模型在量子推理任务上与闭源大模型性能相当，验证了「在 RL 内嵌可验证的规则化反馈」是一条对纯扩规模的高参数效率替代路径。
方法要点：混合验证协议 = 确定性求解器验证 + 语义审查；VRM 做自适应融合 (r = α·r_det + (1-α)·r_sem, α 动态)；RLVR 训练。
启示：这是 RLVR 向 STEM 严谨领域扩展的关键一步——不同于数学题的「答案唯一可验」，量子物理需要「解法约束 + 语义合理」双通道，为后续 STEM-RLVR 工作提供了模板。

KnowRL: Boosting LLM Reasoning via RL with Minimal-Sufficient Knowledge Guidance

arXiv: 2604.12627 · RLVRCurriculumReasoning RL
核心问题：RLVR 在难题上常因稀疏奖励出现梯度为零的「全错轨迹坍塌」。已有「提示式 RL」（Hint-GRPO）试图通过给 partial hint 缓解，但手工设计提示又不可扩展。 方法：KnowRL 将「提示设计」定式化为最小充分知识子集（minimal-sufficient KP subset）问题——把知识点原子化，再用 Constrained Subset Search (CSS) 选出：足够引导学习 & 不过度透题、保留探索空间。 结果：在多跳 QA 与数学推理上超越 Hint-GRPO 与标准 RLVR，尤其在硬题上缓解零梯度坍塌。
方法要点：KP 原子化 → CSS 子集选择 → 将子集作为提示并入 GRPO 的 rollout 初始上下文。
启示：承接本周「RL 只能稳定不能扩能」的讨论（2604.06628）——KnowRL 提供了明确的工程路径：以子集搜索+课程学习替代纯「提高难度」的做法。

Improving LLM Code Reasoning via Semantic Equivalence Self-Play with Formal Verification

Antonio Valerio Miceli Barone, Poon Tsz Nok（爱丁堡大学）
arXiv: 2604.17010 · Code RLSelf-PlayFormal Verification
核心思路：把 Liquid Haskell 形式化证明作为「等价性验证器」，把执行式反例作为「非等价性验证器」，从而让 generator（生成变换后程序）与 evaluator（判断是否语义等价）在自对弈中对抗训练。RL 顶层：把 generator 视作 RL agent，鼓励其规避常见语义错误类。 资产：发布 OpInstruct-HSx——约 28k 条经过验证的合成 Haskell 程序。 结果：在 EquiBench 上最高 +13.3pp 准确率提升；在 PySecDB 上也有一致增益，显示出跨语言的泛化。
方法要点：Liquid Haskell refinement type + 执行式反例 → 验证信号；难度自适应课程；Self-Play 提供无限数据。
启示：把「编译器/证明器」当作训练时 verifier 的趋势在延续（对照 MedVR, OOM-RL）——本作是少有把 全自动形式化证明器 直接用作 RL 验证的尝试。

MT-GRPO: Multi-Turn RL for Tool-Calling Agents with Iterative Reward Calibration

arXiv: 2604.02869 · Tool-Use RLGRPOMulti-Turn
核心贡献：首次将 MT-GRPO（Multi-Turn Group Relative Policy Optimization）与 GTPO 组合，在真实客服任务场景上训练多轮工具调用 agent。核心创新在于 Iterative Reward Calibration——在训练轮次间逐步精化中间奖励。 结果：相比单轮 GRPO，多轮基准显著提升，长水平信用分配问题得到有效缓解。
启示：真实业务的 tool-calling RL 数据稀疏、奖励稀疏、horizon 长，MT-GRPO 给出第一个可工程化的多轮 GRPO 方案。与 2604.14267 (CW-GRPO) 一起构成「把 GRPO 真正推到多轮」的两条并行路线（一个做「轮级 contribution」，一个做「迭代校准」）。

三、Agentic RL 与环境共演化 · Agentic RL & Co-Evolution

CoEvolve: Training LLM Agents via Agent-Data Mutual Evolution

来自 AMAP · Alibaba Group
arXiv: 2604.15840 · Agentic RLSelf-EvolvingData Synthesis
问题：LLM agent 的 RL 训练通常在静态数据分布上进行，无法跟上 agent 能力演进，复杂环境交互覆盖率差。 方法：CoEvolve 建立一个闭环——从 rollout 轨迹中抽出遗忘信号与不确定性信号，识别易出错的交互模式；以这些模式反向驱动 LLM-based 任务合成；合成任务通过环境交互验证后并入训练数据分布。agent 与 data 同步演化。 结果：在 AppWorld 与 BFCL 上，Qwen2.5-7B/Qwen3-4B/Qwen3-30B-A3B 分别取得 +19.43% / +15.58% / +18.14% 绝对增益。
方法要点：rollout → 失败模式抽取（遗忘 + 不确定性） → 任务合成 → 环境验证 → 数据分布更新 → RL 训练 → rollout（回到起点）。
启示：继 RAGEN-2（Template Collapse 诊断）和本作，我们正在进入一个「静态数据 + 静态 verifier + 静态环境 + RL 优化」假设彻底瓦解的阶段——训练循环本身被 RL 化。

AutoSearch: Adaptive Search Depth for Efficient Agentic RAG via RL

Jingbo Sun, Wenyue Chong, Songjun Tu 等（CAS / Meituan）
arXiv: 2604.17337 · Agentic RAGEfficiencyRL
问题：Agentic RAG 常有冗余搜索步，带来显著的计算与延迟代价。作者定义「最小充分搜索深度」——由问题复杂度与 agent 能力共同决定的精度—效率拐点。 方法：在 RL 中每一步用「自生成中间答案」衡量是否已达到最小充分深度；超过则惩罚、未达则奖励。 结果：多跳 QA 基准上显著减少搜索步数，同时精度持平。
启示：和 GUARD（2604.14528）的「entropy-guided short-horizon branching」形成互补——GUARD 在分支维度控成本，AutoSearch 在深度维度控成本。

ClawEnvKit: Automatic Environment Generation for Claw-Like Agents

arXiv: 2604.18543 · Agent InfraEnv GenBenchmark
动机：训练和评测 claw-like agent（具备「抓取-放置」一类低层动作原语）所需的环境长期依赖人工构建，不可扩展。 系统：ClawEnvKit 是一个从自然语言描述自动生成环境的管线，由 parser / generator / validator 三部分构成，显式强制可行性、多样性与结构一致性。 产物：Auto-ClawEval——24 类共 1,040 个环境，与人工构建环境相当/更好，却只花了 1/13,800 的成本。在 4 个模型族 × 8 种 agent 框架下，harness 工程化带来最高 +15.7pp 的性能提升。
启示：和 CoEvolve 的「数据生成」正交——CoEvolve 生成任务，ClawEnvKit 生成环境。两者组合指向一个全自动的 agentic RL 训练工厂。

π-Play: Multi-Agent Self-Play via Privileged Self-Distillation without External Data

arXiv: 2604.14054 · Multi-AgentSelf-PlayDistillation
思路：π-Play 在多智能体自对弈中引入「特权教师」——教师观察双方内部状态，学生仅能观察可见交互；教师向学生蒸馏提供对稀疏奖励 / 合作—对抗任务的密集监督。 结果：无需外部数据，即可在多智能体基准上超越标准 self-play RL 基线，探索出一条「能力自增长 + 零数据依赖」的路线。
启示：对照 MARS²（2604.14564）的共享树搜索多智能体，π-Play 用蒸馏做 credit assignment，是两条思路的分化。

LAnR: Latent Abstraction for Retrieval-Augmented Generation

Ha Lan N.T, Minh-Anh Nguyen, Dung D. Le
arXiv: 2604.17866 · Agentic RAGLatent Reasoning
核心贡献：现有 RAG 依赖在每一跳生成自然语言查询，并强行保持 retriever 与 generator 的架构分离。LAnR 把编码、检索、生成全部合并到一个 LLM 的潜空间中——不再生成文本查询，而是从 [PRED] token 的隐状态直接产出密集检索向量，去匹配同构编码的文档向量。 意义：这等于把「查询生成」从 agent loop 里彻底去掉，retriever 与 generator 变成潜空间的两个模式。对任何搜索型 RL agent（SearchR1, WebAgent 等）都是一次底层重写。
启示：呼应昨日的 Latent CoT（2604.15726）——reasoning 与 retrieval 都在向 latent-state 迁移，surface text 作为中间媒介的历史作用正在消解。

四、对齐理论与 RLHF · Alignment Theory & RLHF

Demystifying the Unreasonable Effectiveness of Online Alignment Methods

Enoch Hyunwook Kang（University of Washington）
arXiv: 2604.17207 · RLHF TheoryOnline DPO
问题：纯贪心更新的迭代式对齐方法（online RLHF / online DPO）在实践中效果显著，但现有的 O(log T) KL-正则化 regret 界显得对经验效果过于悲观。 观点：作者指出问题出在 regret 度量本身——KL-正则化 regret 把「统计学习成本」与「为了探索而软化采样策略的随机性成本」混在了一起。 结论：改用传统的「零温度 regret」（只评估推理时 top-ranked response），作者证明标准的贪心 online 对齐方法（包括 online RLHF 与 online DPO）取得 常数 O(1) 累计 regret——为「为什么这些简单算法效果这么好」给出第一份理论解释。
启示：类似 STaR / DPO 等「贪心自举」方法在理论界长期被认为不如 policy gradient 稳健；这份工作首次在「决策中心」意义下给出与实践一致的理论 endorsement。

HorizonBench: Long-Horizon Personalization with Evolving Preferences

Shuyue Stella Li, Bhargavi Paranjape, Kerem Oktar 等（UW / Meta / OpenAI / Stanford）
arXiv: 2604.17283 · PersonalizationLong ContextBenchmark
问题：真实用户的偏好会随「生活事件」而改变——一个既定偏好在几个月后可能被推翻。评估 LLM 对这类动态信念更新的理解需要长上下文、多事件的对话数据。 基准构造：HorizonBench 包含 4,245 个评测项，取自 360 位模拟用户的 6 个月对话史（平均约 163K token）。数据生成器在结构化心智状态图上演化——生活事件通过类型化依赖边触发偏好改变。 结果：跨模型族的评估把「信念更新失败」识别为一个可定位、可针对性优化的缺陷；同时为长上下文建模、记忆增强架构、心智理论推理提供联合测试床。
启示：和最近的 APEX-MEM / MIA 等记忆增强 agent 工作配对——HorizonBench 给出评测侧的标杆，使得「个性化 RLHF」不再只有 Arena-Bench 式静态偏好。

On the Rejection Criterion for Proxy-Based Test-Time Alignment

arXiv: 2604.16146 · Test-Time AlignmentBoN
贡献：为 proxy RM 驱动的推理期对齐给出原则化的拒绝准则，显式刻画「拒绝试次 vs 对齐增益」的 Pareto 前沿。相较固定阈值的朴素 BoN，在 RLHF 基准上取得更优的对齐—效率边界。
启示：与 Flexible Empowerment BoN（2604.15614）、RCFG（2604.15577）一起，推动推理期 RL 从 heuristic 工程走向理论化。

arXiv: 2604.09676 · TheoryGRPOEntropy

启示：和 LongAct 「激活通道 RL」（2604.14922）呼应——把 RL 的更新限制在模型内部真正参与决策的子空间，是本月一条清晰的效率提升主线。

六、多智能体与多模态 · Multi-Agent & Beyond

Do LLM-derived Graph Priors Improve Multi-Agent Coordination?

Nikunj Gupta, Rajgopal Kannan, Viktor Prasanna
arXiv: 2604.17191 · MARLLLM Prior
问题：LLM 能否基于智能体观测的自然语言描述，生成「协调图先验」供 MARL 使用？作者把这类先验通过图卷积层嵌入 GNN-based MARL 流程。 结果：在 Multi-Agent Particle Environment 四个合作场景下，对比从独立学习到 SOTA 图 MARL 的 baseline，LLM 先验显著提升了协调与动态环境适应性——而且仅需 1.5B 参数规模的 LLM 即可生成有效先验。
启示：为「LLM 辅助的传统 MARL」提供首份定量证据。对「是否要超大模型做 prior」给出否定答案——小模型足以。
