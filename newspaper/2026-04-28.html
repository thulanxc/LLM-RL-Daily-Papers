I · 头版头条 · Headline

SFT-then-RL Outperforms Mixed-Policy Methods for LLM Reasoning

Alexis Limozin, Eduard Durech, Torsten Hoefler, Imanol Schlag, Valentina Pyatkin · ETH AI Center / ETH Zürich / EPFL / AI2 · arXiv:2604.23747

背景：过去一年，「混合策略优化」（mixed-policy optimization）成为 LLM reasoning 训练的热门方向：LUFFY、ReLIFT、Prefix-RFT、SRFT 等方法在 SFT 与 RL 信号之间动态切换或加权混合，普遍声称比经典「先 SFT 再 RL」两段式管线更强。

核心发现：作者复现这些工作时定位到两个被忽视的实现缺陷：(1) DeepSpeed CPU-offloaded optimizer bug——在 gradient accumulation 时静默丢弃中间 micro-batch；该 bug 已渗透到 TRL、OpenRLHF、Llama-Factory 等下游框架；(2) OpenRLHF loss aggregation bug——以错误权重聚合 per-mini-batch loss。两个 bug 共同压低了 SFT 基线，使 mixed-policy 看起来「赢得很轻松」。

修正结果：修复后，Qwen2.5-Math-7B 上 SFT-then-RL 比 SRFT 高 +3.8 分，Llama-3.1-8B 上更高 +22.2 分。在 in-distribution 任务上，仅修正后的 SFT 单段（52.2）已超过 LUFFY (46.3)、ReLIFT (48.8)、Prefix-RFT (51.8)；再叠加 RL 后均值跃至 57.0。

方法学贡献：论文不仅给出修复补丁与重新评估，还系统化呼吁社区在每篇 RL post-training 论文中报告所用框架的精确 commit 与 optimizer 配置，并把 SFT 基线作为强 anchor 而非弱 baseline。

关键启示：
  • 过去半年「mixed-policy 优于 SFT→RL」的故事在很大程度上需要重写。
  • RL 工程栈的隐性 bug 可以让 baseline 性能差距达 20+ 分，超过任何算法创新。
  • SFT 阶段的 dense reward 信号对后续 RL 至关重要，不应被视为「过时」。

II · 奖励建模 · Reward Modeling

Reward Models Are Secretly Value Functions: Temporally Coherent Reward Modeling

Alex Nikulkov · arXiv:2604.22981

核心洞察：标准 RLHF 中 reward model 仅在 final token 上输出标量，丢弃了对中间位置的丰富信号。论文论证：一个训练良好的 reward model，在任意 token 处的输出本应代表「给定当前响应前缀，最终 reward 的条件期望」——这正是价值函数的定义。

方法：TCRM（Temporally Coherent Reward Modeling）在标准 Bradley-Terry 损失之上增加两项正则：(1) Monte-Carlo 一致性项——中间 token 的预测应逼近最终 reward；(2) Temporal Difference 项——相邻 token 之间的预测应保持平滑过渡。这相当于让 reward model 同时学习 V 函数。

关键结果：(1) 可解释性——中间 token 的 pairwise accuracy 由 50%（随机）跃升到 88.9%，可视化得到 token 级 reward trajectory；(2) SOTA PRM——ProcessBench 平均 F1 44.9%；(3) 统一 reward / value——PPO 训练时不再需要单独的 critic，显存峰值降低 27%、step time 降低 19%，与 critic-free 方法（如 GRPO）相比仍保留逐 token 价值。

关键启示：
  • 把 reward model 视作 value function，是 PRM 与 PPO critic 的统一视角。
  • Bradley-Terry 之外加 MC + TD 正则，是用 dense 信号「无监督」获得过程奖励的轻量做法。
  • 对训练栈的工程意义：可省掉单独 critic，加速 PPO。

K-Score: Kalman Filter as a Principled Alternative to Reward Normalization in RL

Sumxia (匿名通讯) · arXiv:2604.23056

问题：策略梯度方法普遍依赖经验式 reward normalization（running mean/std），在非平稳奖励环境下不稳定。GRPO 的 group-mean baseline 同样面临这一问题。

方法：引入1D Kalman filter对 latent reward 均值（与方差）进行在线估计；将其用作 advantage 预处理。开销极小（一个标量滤波器），无需修改 policy 网络。

实验：LunarLander、CartPole 上 REINFORCE / Actor-Critic / GAE / PPO 均加速收敛、降低 training variance。论文与并发工作 KRPO（将 Kalman 应用于 GRPO baseline）形成互补：K-Score 关注「通用 RL 的 advantage 标准化」，KRPO 关注「LLM reasoning 中的 group baseline」。

关键启示：把 reward 估计当作经典在线滤波问题，是替代 running-mean 的更原则化做法；对 LLM RL 而言，可在 GRPO/RLOO 上无缝替换 group baseline。

COSPLAY: Co-Evolving LLM Decision and Skill Bank Agents for Long-Horizon Tasks

Xiyang Wu et al. · arXiv:2604.20987

动机：LLM 在长程任务中常因缺乏「跨 episode 复用结构化技能」的机制而表现不一致。

方法：COSPLAY 协同演化框架——(1) 决策 agent：每步从可学习的 skill bank 检索技能以指导动作选择；(2) skill pipeline：从未标注的 rollout 中自动发现可复用技能并归档进 bank。两者通过迭代 RL 联合演化：决策 agent 用更好技能产生更好 trajectory，pipeline 在更好 trajectory 中提取更优技能。

实验：跨 6 个游戏环境，COSPLAY 用 8B 基模就实现单玩家任务 +25.1% 平均奖励提升，超过 4 个 frontier LLM baseline；多人社交推理游戏上保持竞争力。代码与项目页公开。

关键启示：「skill bank + decision agent 的双向 RL 协同演化」是把短程 RL 提升到长程任务的有效抽象，避开单一 token-level RL 难以处理的稀疏奖励。

CoFi-PGMA: Counterfactual Policy Gradients under Filtered Feedback for Multi-Agent LLMs

Stela Tong, Elai Ben-Gal · arXiv:2604.22785

问题设定：多 LLM 协作系统（路由竞争 / 协同生成）只能观察「最终输出」的反馈，而不知道每个 agent 的边际贡献——这是经典 multi-agent credit assignment 在 LLM 场景的再现。

关键启示：多 agent LLM 不再「只能整体打分」——counterfactual baseline 把信用分配从黑盒变成可估计量。

RouteGuard: Internal-Signal Detection of Skill Poisoning in LLM Agents

arXiv:2604.22888

威胁模型：当 LLM agent 通过外部「skill / tool」库扩展能力时，恶意 skill 描述可以悄悄诱导 agent 偏向恶意工具——这是一种新型间接 prompt injection。论文将其命名为「skill poisoning」。

关键观察：成功投毒会触发「attention hijacking」——响应阶段的注意力从 trusted context 转向恶意 skill span，留下可检测的内部信号。

方法：RouteGuard——冻结 backbone，通过响应条件下的 attention 与 hidden state alignment 做可靠性门控（reliability-gated late fusion）的二分类检测。

关键启示：Agentic RL 不只是要训练「更强 agent」，也要训练「能识别工具被污染的 agent」；attention pattern 是廉价高信号的探针。

KARL: Mitigating Hallucinations in LLMs via Knowledge-Boundary-Aware RL

arXiv:2604.22779

出发点：「让模型该弃答时弃答」是缓解幻觉的最直接路径，但与模型的知识边界动态绑定——边界随训练进展不断扩张／收缩。

方法：(1) Knowledge-Boundary-Aware Reward：根据样本是否在模型当前能力边界内，动态决定「正确作答 / 弃答 / 错误」的奖励权重；(2) 两阶段 RL：阶段一探索知识边界，阶段二把错误回答转化为弃答（abstention）。这一设计避免了静态 abstain 标签所带来的 over-refusal。

意义：把「不知道」当成可学习行为，而不是 prompt 工程的副产物；与 RLHF safety 训练正交，可叠加。

关键启示：RL 对齐不仅可以塑造「答得更好」，也可以塑造「该不答时不答」；用动态奖励跟踪知识边界比固定标签更稳健。

Aishwarya Padmakumar, Leon Derczynski, Traian Rebedea, Christopher Parisien · NVIDIA · arXiv:2604.23067

现状：当前自动化红队工作几乎都绑定在「内容安全 / moderation」目标上——红队模型针对一个内容审核模型做对抗优化。这导致两个问题：(1) 训练时需要预存评估器；(2) 在新对抗目标（提示注入、越权、隐私）上不可迁移。

When Policies Cannot Be Retrained: Closed-Form Post-Training Steering for Offline RL

arXiv:2604.22873

问题：很多产品场景下 actor 训练完便冻结，但部署时目标可能漂移（数据分布变化、合规要求变化、多 SKU 切换）。如何「在不重训」的前提下做 post-training steering？

主要结果：对角高斯 actor / prior 的设定下，论文给出闭式定理：Product-of-Experts 组合（系数 α）等价于 KL 正则适配（β = α/(1−α)）下的同一确定性策略，仅协方差差一个全局标量。这把多种 deployment-time steering 方法统一在一条公式下。

实验：4 个 D4RL 环境、3,900 MuJoCo 回合，在退化 / 随机 prior 下，precision-weighted 组合保持锚定到原 actor，而 additive / prior-only 适配会崩溃；KL-budget selector 常能恢复到接近 oracle 的工作点（HELP/FROZEN/HURT 比例 4/5/3）。

关键启示：对 LLM RL 落地有直接指导意义——RLHF 模型部署后再做 inference-time alignment / steering 时，PoE 与 KL 调节并非独立技巧，而是同一闭式族。

CODA: Coordination via On-Policy Diffusion for Multi-Agent Offline RL

arXiv:2604.23308

问题：离线多 agent RL 训练时常见「coordination failure」——agent 各自最优但联合策略糟糕。本文将其视作「联合 trajectory 与 joint policy 不匹配」。

方法：CODA——基于扩散模型的多 agent 联合 trajectory 生成器，采样时以当前 joint policy 为条件，使生成的 trajectory 与 policy 同步演化（co-adapt）。这把扩散模型从「数据增强」升级为「协调机制」。

与 LLM RL 的关系：当 LLM 作为多 agent（router、specialist、verifier）协作时，CODA 的「on-policy 扩散」思路可以借鉴——以扩散 prior 替代 GRPO 中的 group-mean 基线，实现策略间协调。

关键启示：扩散模型不仅可以生成 reasoning，也可以做多 agent 协调；与 token-level RL 形成互补。

Prism-Reranker: Beyond Relevance Scoring — Joint Contributions and Evidence for Agentic Retrieval

Dun Zhang · arXiv:2604.23734

问题：RAG / agent 系统中，下游消费者需要的不是简单 relevance 标量——它们要决定「这段是否有用、为什么有用、能否裁剪」。传统 reranker 只输出分数，迫使 agent 把整篇文档塞进 context。

方法：Prism-Reranker 基于 Qwen3.5（0.8B / 2B / 4B / 9B 四档）。每次判断不仅给出 yes/no relevance，当判断为 yes 时额外输出：(1) contribution statement：本文档对查询的具体贡献；(2) evidence passage：保留所有 query 相关信号、剔除噪声的自包含改写。训练采用 hybrid 目标——从强商业 reranker API 做 point-wise 蒸馏 + 在 contribution / evidence 标注上做 SFT。

为何收录：虽然主体是 SFT + 蒸馏，但其设计直接服务于 agentic RL 的检索调用（reduce context、提供 evidence 给 verifier 用），是「让 retriever 服务于 RL 训练的 agent」的代表性工作。

关键启示：RAG 中 reranker 的角色应升级为「agent 友好」——给出可裁剪的 evidence，从而让上游 agentic RL 训练时获得更稳定 reward 信号。

C-MORAL: Controllable Multi-Objective Molecular Optimization with Reinforcement Alignment for LLMs

Rui Gao, Youngseung Jeon, Swastik Roy, Morteza Ziyadi, Xiang 'Anthony' Chen · UCLA / Amazon · arXiv:2604.23061

背景：药物分子设计要求模型在多个互相竞争的属性（活性、毒性、ADMET 等）间做可控折中——这是典型的 multi-objective alignment 问题。

方法：C-MORAL 是基于 RL 的 LLM 后训练框架，三大组件：(1) group-based relative optimization（GRPO 风格组采样优化）；(2) 跨异质目标的 property-score alignment（把不同尺度属性映射到同一相对分布）；(3) 连续非线性 reward aggregation（处理 trade-off 而非简单加权和）。

关键启示：把 multi-objective alignment 当成 GRPO 的扩展——而不是单独的 Pareto 算法——是把通用 LLM RL 工程移植到科学领域的可行路径。
