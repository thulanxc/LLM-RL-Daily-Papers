"All the RL that's fit to read" — 大语言模型强化学习每日速览

编者按 · 今日要闻

今日共收录 14 篇 全新 LLM 强化学习相关论文，主题在过去 24–48 小时内呈现出三条清晰的主线： (1) 推理 RL 从"外部验证"走向"模型自证与弱监督" —— DDRL（2604.21327）系统拆解了 Test-Time RL (TTRL) 中"伪标签歧义区"如何经 GRPO 组相对优势放大为虚假信号，提出频率采样 + 去偏优势双修复，可在多数学基准上一致地跑赢原 TTRL； GRPO-VPS（2604.20659）不依赖任何外部 PRM/ORM，只用模型自身对正确答案的隐式置信变化作为"可验证过程监督"，将 GRPO 的盲目组内归因细化到步级； Weak-Supervision RLVR（2604.18574）跨三种弱监督设定（稀疏/噪声/自监督代理）系统地识别出"推理忠实性"才是决定能否泛化的前置指标，给"RL 前该先做什么 SFT"提供了新的工程判据； VPS Verbal Critique（2604.21611）则把"更强教师的口头批评"作为训练-free 的过程奖励，在 GPQA Diamond 上取得 94.9% 的新 SOTA。

(2) Agentic RL 的"系统-数据-结构"协同进化 —— AgenticQwen（2604.21590）给出工业级小模型 Agent 的双飞轮配方：推理飞轮按错误加难度，Agentic 飞轮把线性工作流扩展成多分支行为树，实现"小模型追平大模型"的搜索/数据分析工业部署； AEL（2604.21725）把 Agent 视作"快慢双时标耦合系统" —— 快尺度用 Thompson Sampling 选记忆策略，慢尺度用 LLM 反思动态生成新的检索策略，Sharpe Ratio 在连续 208 集的资产组合任务上冲到 2.13； TPGO / GRAO（2604.20714）把多 Agent 系统建模为文本参数图，用"文本梯度"执行组相对 Agent 优化，首次让"优化器本身"也随历史一同进化； DiffMAS（2604.21794）则跳过自然语言接口，让 Agent 通过 KV-cache 潜通信联合优化推理与通信。

(3) "跨域泛化"成为新瓶颈，多语言 / 多编程语言 / 小-大模型协作是今日关键词 —— polyGRPO（2604.21593）发现非英语回复在推理任务上反而更准，首次把语言本身作为探索潜变量，仅用 18K 多语数学问题就把 Qwen2.5-7B 的英文基准提升 6.72 pp； Parallel-SFT（2604.20835）揭示"源语言 RL 训练竟会损害目标语言"的反直觉现象，并以平行代码 SFT 初始化撬动跨 PL 零样本迁移； Dynamic SLM↔LLM（2604.17827）用在线 RL 训练小模型"何时求助大模型"，发现强 SLM 更自主、强 LLM 使交互更稀有且信息更浓；最后， 红队 RL（2604.21159）把自动红队从"模版化组合"升级为有效性 × 多样性联合优化，而 IRM（2604.21223）直接利用基础模型与对齐模型之间的隐式奖励差分检测 AI 文本，无需任何偏好标注或微调。

I · 推理 RL / RLVR — 从外部验证走向模型自证

DDRL: 理解与抑制 Test-Time RL 中的伪信号放大

Yongcan Yu, Lingxiao He, Jian Liang, Kuangpu Guo, Meng Wang, Qianlong Xie, Xingxing Wang, Ran He
arXiv: 2604.21327 · 2026-04-23

核心问题 Test-Time RL (TTRL) 在推理期用自一致伪标签进行 GRPO 训练，但对标签噪声高度敏感 —— 尤其是中等一致性的"模糊样本"既非正确多数、也非少数错，却经常被 group-relative advantage 错误放大为强信号，导致训练越跑越偏。

方法 作者提出统一框架 DDRL（Debiased and Denoised test-time RL）：

  • 频率采样：识别并排除"模糊一致性"区间的样本，同时保持正负样本平衡；
  • 去偏优势：使用固定优势替代组内均值归一化，消除 GRPO 对尺寸与多数比例的隐偏。

结果 在 3 个主流 LLM × 多个数学推理 benchmark（AIME/MATH-500/Olympiad/MinervaMath 等）上，DDRL 一致超过 TTRL 原版，并对噪声伪标签显示明显更好的鲁棒性。

意义 把 TTRL 的失败模式从"幻觉标签"具体刻画为"模糊域样本 × 组优势放大"这一双因子，并给出可直接替换的修补件 —— 对所有使用自一致伪监督的 online RL 框架都具参考价值。

GRPO-VPS: 用可验证过程监督增强组相对策略优化

Jingyi Wang, Lei Zhu, Tengjin Weng, Song-Li Wu, Haochen Tan, Jierun Chen, Chaofan Tao, Haoli Bai, Lu Hou, Lifeng Shang, Xiao-Ping Zhang
arXiv: 2604.20659 · 2026-04-22

核心问题 标准 GRPO 依赖仅结果的可验证奖励，同一轨迹内的每一个 token 被赋予相同优势，使"早期关键决策"与"后期琐碎展开"获得相同的信用，信号稀疏且噪声大。

方法 GRPO-VPS 提出一种无需额外 PRM / 外部模型的可验证过程监督思路：

  • 将生成轨迹按语义切段，作为候选"过程步"；
  • 在每段边界处探测模型对正确答案的条件概率，其变化视为"本段是否有效推进"的可验证信号；
  • 将该信号直接注入 GRPO 的优势估计，实现步级信用分配，而策略本身仍无需额外打分器。

结果 在多个推理基准上，GRPO-VPS 显著优于纯结果奖励 GRPO 与多种 PRM-GRPO 变体，并保持训练开销与原 GRPO 几乎一致。

意义 把"过程奖励"从外挂打分器拉回到策略模型自己对答案的信念流，是 RLVR 在"零额外标注"下做步级信用分配的少见实现路径，具直接落地价值。

Process Supervision via Verbal Critique Improves Reasoning in LLMs

Hao-Yuan Chen
arXiv: 2604.21611 · 2026-04-23

核心问题 过程奖励模型（PRM）训练成本高、泛化差；本文探索一个简单问题：一个更强模型的口头批评，是否足以替代显式 PRM 成为"过程信号"？

方法 提出 Verbal Process Supervision (VPS) —— 一个完全 training-free 的迭代 "generate→critique→refine" 循环：actor 模型（可以较弱）产出候选推理链，supervisor 模型用结构化自然语言给出步级批评，actor 依此改写；循环至多 R 轮。作者系统性地对比了批评粒度（答案级 vs. 步级）、supervisor 强度与 budget R。

结果

  • GPQA Diamond 上 R=4 时达到 94.9%，超过此前 94.1% 的 SOTA（且不做任何梯度更新）；
  • AIME 2025 上把弱 actor 的分数从 11.7-26.7% 拉升到 63.3-90.0%，最多 +63.3 pp；
  • 在匹配计算量下，VPS 比 Reflexion 提高 +8.5 ~ +12.1 pp，比 Self-Consistency@5 提高 +5.0 ~ +8.3 pp。

意义 作者论点鲜明："批评粒度"才是关键变量，而不是"是否有口头反馈"。这对当前许多"训练过程奖励模型"的工作构成严肃挑战 —— 一个足够细粒度的自然语言教师可能就是最好的过程奖励。

When Can LLMs Learn to Reason with Weak Supervision?

Salman Rahman, Jingyan Shen, Anna Mordvina, Hamid Palangi, Saadia Gabriel, Pavel Izmailov
arXiv: 2604.18574 · 2026-04-20

核心问题 随着模型能力提升，高质量可验证奖励越来越难构造。作者系统性地问：在三种弱监督（稀疏数据 / 噪声奖励 / 自监督代理奖励）下，RLVR 究竟何时能成功？

方法 跨模型家族（Llama3/Qwen2.5/Mistral 等）、跨推理域（数学、科学、常识）做大规模实证扫描。关键测量包括训练奖励饱和曲线、推理忠实性（intermediate steps 对最终答案的逻辑支持度）、输出多样性等。

关键发现

  • 饱和动力学预测泛化：成功泛化的模型存在明显的"预饱和期"；快速饱和者只是记忆而非学习。
  • "推理忠实性"是关键前置指标，而单纯的输出多样性几乎无预测力。
  • 修复路径：在推理轨迹上做 SFT 是弱监督下泛化的必要条件；在域数据上做持续预训练能放大这一效果。两者联合应用到 Llama3.2-3B-Base 上，使其在三种弱监督设置下全部成功泛化（此前全部失败）。

意义 该工作给出了一个可操作的"RL 前诊断"框架：先测推理忠实性 → 决定是否需要补 SFT + CPT → 再进入 RLVR，而不是盲目堆 RL 步数。对当下"R1-Zero 式端到端 RL"路线是有力的补充与纠偏。

polyGRPO: 将语言作为推理优化的潜变量

Linjuan Wu, Haoran Wei, Jialong Tang, Shuang Luo, Baosong Yang, Yongliang Shen, Weiming Lu
arXiv: 2604.21593 · 2026-04-23

核心观察 随着 LLM 去英语中心化，一个反直觉现象出现：非英语回复在某些推理任务上反而比英文更准。作者假设：语言不仅是表面形式，而是一个结构性调制内部推理路径的潜变量。

方法 提出 polyGRPO，将"回复语言"的变异显式视为 GRPO 的探索信号：

  • 对同一问题采样多种语种的推理链（polyglot thinking），让语言选择成为 group-relative advantage 的一维探索轴；
  • 奖励同时考虑答案正确率与推理格式保真度；
  • 整条流程只用 18.1K 多语数学题，无需任何 CoT 标注。

结果

  • Qwen2.5-7B-Instruct: 英文推理测集 +6.72 pp，多语基准 +6.89 pp；
  • 是唯一能在只用数学训练数据的情况下把英文常识推理 (X-CSQA) 提升 +4.9 pp 的方法；
  • 在 MGSM / MATH500 / PolyMath-medium / X-CSQA 共 4 任务 × 23 语言上一致有效。

意义 给"多语能力是副作用还是潜空间"这一长期争论加入了关键证据：语言差异本身是极低成本的高价值探索维度。对多语 RL、认知诱导 (cognitive elicitation) 以及未来的思维链语言选择都具启发。

II · Agentic RL — 系统、数据与结构的协同进化

AgenticQwen: 双数据飞轮驱动的小型 Agentic 语言模型训练

AgenticQwen Team (Qwen)
arXiv: 2604.21590 · 2026-04-23

核心目标 工业场景下 Agent 模型需要满足严格的成本与延迟约束，因此小模型的多步推理与工具使用能力成为关键瓶颈。

方法 AgenticQwen 基于 GRPO 式多轮 RL，在合成数据 + 少量开源数据上做推理 RL 与 Agentic RL 并行训练，并配以双数据飞轮：

  • 推理飞轮：通过从错误中学习来持续抬升任务难度，生成涵盖数学、搜索、代码解释器等多工具调用的长步推理；
  • Agentic 飞轮：把线性工作流扩展为多分支行为树，更真实地反映用户交互中的决策复杂度；
  • Agentic RL 阶段使用rubric-based evaluator对任务分解为可验证子目标，给出 0/1 密集奖励。

结果 AgenticQwen 在多个 Agentic benchmark 上取得强结果，其工业搜索 / 数据分析系统上，小模型版本显著缩小与更大模型的差距，在真实部署环境中已稳定服务。

意义 这是目前公开最完整的"工业级小 Agent"训练配方之一 —— 同时回答了"数据从哪来、奖励如何设计、飞轮如何自扩"三个关键工程问题，对中小规模团队训练可部署 Agent 具有直接参考价值。

AEL: Agent Evolving Learning for Open-Ended Environments

Wujiang Xu, Jiaojiao Han, Minghao Guo, Kai Mei, Xi Zhu, Han Zhang, Dimitris N. Metaxas (Rutgers)
arXiv: 2604.21725 · 2026-04-23

核心问题 现有 LLM Agent 即便已具备 planner、tool 与 memory，仍然基本是无状态的：每个 episode 都从零开始解，过去的经验并不稳定转化为更好的未来决策。

方法 作者把 Agent 当作一个快慢双时标耦合系统：

  • 快尺度（每 episode）：Thompson Sampling bandit 在若干候选记忆检索策略间选择，本质上是对"何时取什么经验"做在线概率学习；
  • 慢尺度（失败诊断期）：由 LLM 驱动的反思模块分析失败模式，把因果洞见注入决策 prompt；当诊断判断仅调 prompt 不够时，框架会主动生成新的检索策略加入候选池；
  • 三者共同组成可进化系统：memory、policy bandit、reflection 全部可学。

结果 在 10 支分行业股票 × 208 episodes × 5 seed 的顺序投资组合基准上，AEL 达到 Sharpe Ratio 2.13，显著优于静态 Agent 与单一记忆机制对照。

意义 把"Agent 进化"这一常被泛化为口号的概念，落到了"记忆策略"作为一级可学元素、反思作为结构生成器的具体框架，为长期运行的 AI 员工 / 决策代理提供了新的设计范式。

TPGO / GRAO: 自改进多 Agent 系统的文本参数图优化

Shan He, Runze Wang, Zhuoyun Du, Huiyu Bai, Zouying Cao, Yu Cheng, Bo Zheng
arXiv: 2604.20714 · 2026-04-22

核心问题 多 Agent 系统的自动优化目前多为扁平的 prompt 调优，缺乏结构感知；更关键的是优化器本身通常是静态的，不会从历史经验中学会更好的优化策略。

方法 提出 Textual Parameter Graph Optimization (TPGO)：

  • 把 MAS 建模为文本参数图 —— Agent、工具、工作流均为模块化可优化节点；
  • 引入文本梯度：从执行轨迹中抽取结构化自然语言反馈，定位失败并建议修改；
  • Group Relative Agent Optimization (GRAO)：对 agent 级变化构造组相对优势，作为 meta-learning 策略，使优化器能在跨任务历史上学习。

意义 TPGO 把 GRPO 的"组相对"思想从 token/回答级推广到 Agent / 工具 / 工作流模块级，同时让优化器本身随任务历史一起进化；对正在构建自主工程师 / 分析师 Agent 的团队来说，这提供了把 "系统 = 被训对象" 这一思路完整工程化的第一个统一框架。

DiffMAS: 多 Agent 语言系统端到端的潜通信优化

Ye Yu, Heming Liu, Haibo Jin, Xiaopeng Yuan, Peng Kuang, Haohan Wang (UIUC)
arXiv: 2604.21794 · 2026-04-23

核心问题 当前 MAS 工作大多聚焦"角色与编排"，却把Agent 间通信当作固定的自然语言接口。这造成信息瓶颈与推理—通信解耦的优化劣势。

方法 DiffMAS 把潜通信（例如通过内部 KV-cache / 连续表征）视为可学组件，进行参数高效的监督训练：

  • 在多 Agent 潜轨迹上做差分优化，让 Agent 联合学会如何编码 / 如何解读彼此的内部状态；
  • 保留自然语言接口作为外部可解释通道，但内部协作走潜通路，以提升信息密度；
  • 支持 SFT 初始化后可扩展到 RL 微调。

结果 在数学推理、科学 QA、代码生成与常识 benchmark 上，DiffMAS 一致优于单 Agent 推理与基于文本的多 Agent 基线，且解码稳定性明显提升。

意义 为"语言 Agent 之间是否必须只讲人话"这一常被默认的架构前提提供了反面证据 —— 适度放宽接口能同时提升推理上限与协作效率，对下一代推理框架的通信层设计具有启示意义。

Learning to Seek Help: 小模型与大模型的动态协作

Anonymous (under review)
arXiv: 2604.17827 · 2026-04-19

核心问题 LLM 能力强但成本和隐私代价高，SLM 高效本地可部署但能力受限。如何让 SLM 学会在多步推理中主动、适时地求助 LLM，同时让 LLM 给出自适应反馈而非被动工具调用？

方法 用在线 RL 训练 SLM 的"决策触发 + 查询构造"策略；LLM 侧则提供依据当前对话状态的可变粒度反馈。作者系统性地消融了 SLM 能力、LLM 能力、效率 / 隐私约束对协作策略的影响。

关键发现

  • 清晰的缩放效应：更强的 SLM → 更自主，求助次数下降；更强的 LLM → 求助次数下降但每次携带更多信息；
  • 学得的动态协作策略显著超越静态 pipeline 与单体推理；
  • 协作策略对未见过的 LLM也能稳健迁移。

意义 为"边缘 SLM + 云端 LLM" 这一日渐主流的部署范式提供了在线 RL 的完整训练方法，并首次用经验数据勾勒出 SLM/LLM 能力对协作频次与信息密度的双向影响图谱。

III · 代码 RL · 泛化与迁移

Parallel-SFT: 跨编程语言零样本迁移的 Code-RL 初始化

Zhaofeng Wu, Shiqi Wang, Emma Peng, Anuj Goyal, Melanie Kambadur, Sebastian Ruder, Yoon Kim, Chloe Bi (Meta Superintelligence Labs · MIT)
arXiv: 2604.20835 · 2026-04-22

关键发现 作者首先提出"跨编程语言零样本 Code-RL 迁移"这一任务并给出一个反直觉实证：对 Llama-3.1 在某一源 PL 上做 RL 训练，不但不能改善其他目标 PL 的表现，反而可能使其退化。

假设与方法 作者假设：RL 的迁移前提是一个可泛化的 SFT 初始化。于是提出 Parallel-SFT：

  • 在 SFT 数据中加入平行程序 —— 同一功能在多种 PL 中的等价实现；
  • 使得模型在 SFT 期就学到跨 PL 的语义不变量，而非死背某一语言的 API；
  • 在此之上再跑 Code-RL，即可实现零样本跨 PL 迁移。

结果 Parallel-SFT 初始化后，源 PL 上的 RL 改进能稳定正向迁移到目标 PL；作者给出多对 PL 与多种模型规模下的一致结果。

意义 给"Code-RL 的泛化边界"画出第一条明确等高线 —— 这对多语言代码模型、低资源 PL（Rust / Solidity / Move / OCaml 等）的能力扩展具重要工程含义。

IV · 对齐、安全与检测

Adaptive Instruction Composition: 自动化 LLM 红队的强化学习

Jesse Zymet, Andy Luo, Swapnil Shinde, Sahil Wadhwa, Emily Chen (Capital One · AI Foundations)
arXiv: 2604.21159 · 2026-04-23

核心问题 自动红队通常靠模板化组合攻击文本，面临两难：组合越激进则成功率越高，但多样性骤降，使攻击面偏向少数漏洞。

方法 提出 Adaptive Instruction Composition：

  • 把红队视为在"众包文本组合空间"里的组合型决策任务；
  • 用 RL 联合优化有效性 × 多样性两类指标；
  • 探索/利用机制被显式训练，以在组合空间中搜索针对目标漏洞的"定制化生成"。

结果 在多个有效性与多样性 metric 上全面超越随机组合 baseline，且在模型迁移设置下（换受害模型）仍保持显著优势。

意义 自动红队正从"模版枚举"进入"RL-driven 组合探索"时代；这对防御侧（对抗训练、数据选择、越狱检测）也意味着需要同时提升"多样性鲁棒性"。

IRM: 利用隐式奖励模型零样本检测 LLM 生成文本

Runheng Liu, Heyan Huang, Xingchen Xiao, Zhijing Wu
arXiv: 2604.21223 · 2026-04-23

核心问题 已有 AI 文本检测方法要么依赖大量偏好数据训练判别器，要么需要白盒 logits；对于实际部署场景过重。

方法 作者注意到："指令微调模型 vs. 同族基础模型"之间的 log-likelihood 差分恰好就是一个隐式奖励模型（IRM）（这是 DPO 的核心推论）。IRM 方法：

  • 只使用公开可得的基础模型与其指令微调版；
  • 在候选文本上计算二者的概率差，作为"是否受对齐偏好塑造"的隐式奖励；
  • 完全零训练 / 零标注；也无需偏好数据收集。

结果 IRM 超过此前多种 reward-based 检测器与 likelihood-based 基线，是当前最轻量的检测路径之一（NeurIPS 2025 Poster）。

意义 为"对齐训练本身留下可检测的痕迹"这一理论给出直接实证，也拓展了 DPO 隐式奖励作为模型属性探针的应用面。

V · 策略优化算法 · 理论侧

On Reward-Balancing Methods for Reinforcement Learning

Simone Baroncini, Bahman Gharesifard, Giuseppe Notarstefano
arXiv: 2604.20433 · 2026-04-22

主旨 研究一类"奖励再平衡方法"：通过迭代地调整奖励函数把折扣回报 RL 问题转化为"最优策略是 greedy"的等价问题，从而简化策略求解。

贡献

  • 系统给出涉及变换的代数结构分析；
  • 把 reward-balancing 重述为一个最优控制问题，引入控制论工具进行稳定性与收敛性分析；
  • 扩展到模型不确定性情形 —— 在随机模型采样下给出归一化保证与随机涨落的概率界。

与 LLM 的联系 虽然本文主要在经典 RL 语境下展开，但"奖励再平衡"即把稀疏外部奖励压缩到等价密集奖励这一思想，与当前 LLM RL 中关于奖励稠化 / potential-based shaping / KL-regularized PG 的分析紧密相关，值得推理 RL 研究者参考其控制论工具箱。
