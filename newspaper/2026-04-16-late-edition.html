编辑视角 · Editorial Overview

Agentic RL 的诊断时代到来。本期最引人注目的是 RAGEN-2 对 Agentic RL 中 "模板坍缩" (Template Collapse) 现象的系统诊断。与仅关注熵的传统做法不同，RAGEN-2 首次将推理质量分解为 "输入内多样性"（熵）与 "跨输入区分度"（互信息 MI），并证明 MI 才是与最终性能更相关的代理指标。这一工作标志着 LLM RL 领域从 "盲训" 向 "带诊断训练" 的范式跃迁。

过程奖励 (Process Reward) 全线回归。本期有四篇独立工作围绕过程奖励展开：PROGRS 通过结果条件化居中化 (outcome-conditioned centering) 将 PRM 分数转化为无偏的相对偏好；Process Reward Agents (PRA) 将 PRM 变为可插拔的测试时组件；Rubrics-to-Tokens (RTT) 桥接响应级评分和 token 级信用；SubSearch 则用内在的过程奖励取代外部监督。过程奖励已从 "只适用于数学/代码" 扩展到知识密集型推理、指令遵循、复杂检索等领域。

Tool-Use RL 多点开花。TInR 主张将工具知识内化到 LLM 中以避免文档依赖；TOOLCAD 将 LLM 变为 CAD 建模智能体；AnomalyAgent 用五工具闭环生成工业缺陷数据；COVERT 给出可验证的 tool-use 数据合成流程。这一组工作共同指向：Agentic RL 所需的环境基础设施正快速成熟。

多跳知识推理成为新战场。KG-Reasoner 用 RL 把多步 KG 遍历内化到单一 "思考" 阶段；TRACE 通过叙事与经验先验增强多跳 KGQA 的一致性；AgentGL 将图学习形式化为智能体决策过程；这些工作表明 RL 正成为解决 KGQA 碎片化推理问题的主流路径。

效率导向的 CoT 优化。Graph-Based CoT Pruning 指出 "低价值反思" 和 "重复反思" 两类冗余，用 DAG + 双层剪枝将推理 token 减少 42% 而精度不降。E-GRM 则在生成式奖励建模中引入 "按需 CoT"，通过模型内部不确定性触发思考。CURE 则把 "输出中哪些声明可信" 转化为 claim-level 置信估计。

RLVR 的边界实验。Regime Boundaries 在金融交易域提醒研究者：即使 LLM 提取了有效的 IR(Information Coefficient ≥ 0.15) 信号，在宏观冲击引起的分布漂移下也会拖累下游 PPO 智能体。OOM-RL 反其道而行之，把 "真实资本损失" 作为不可黑客化的负梯度信号，历时 20 个月把 MAS 训练出 2.06 的 Sharpe Ratio。两篇论文共同展示了 RL 在非稳态现实环境中的两面性。

I. Agentic RL · 智能体强化学习

RAGEN-2: Reasoning Collapse in Agentic RL2604.06268

Zihan Wang, Ziqi Pang, Manling Li et al. · Northwestern University · April 7, 2026
Agentic RL Diagnosis Mutual Information Template Collapse
核心贡献：首次对 Agentic RL 中一个被反复观测但未被命名的失效模式——"模板坍缩 (Template Collapse)"——给出系统刻画和可观测的诊断工具。团队发现：即使熵保持稳定，模型也可能退化为依赖"看起来多样但其实与输入无关"的固定模板。熵只能衡量同一输入内的多样性，无法判断推理是否真正对不同输入做出响应。

方法：将推理质量分解为 (1) 输入内多样性 (Entropy) 与 (2) 跨输入区分度 (Mutual Information, MI)，并引入一组轻量 MI 代理指标用于在线诊断。实验表明 MI 与最终性能的相关性显著高于熵，是更可靠的推理质量代理。作者进一步用信噪比 (SNR) 机制解释模板坍缩：低 reward 方差削弱任务梯度，让正则项主导、擦除跨输入的推理差异。据此提出 SNR-Aware Filtering，按每轮的 reward 方差作为轻量代理挑选高信号 prompt。

Key Takeaways

  • 熵 ≠ 推理质量；跨输入互信息是更好的代理
  • 模板坍缩是 entropy-invisible 失效模式，解释了为何"训练曲线看起来健康但性能不涨"
  • SNR-Aware Filtering 只用 reward 方差就能显著缓解，不引入额外训练成本

Skill-SD: Skill-Conditioned Self-Distillation for Multi-turn LLM Agents2604.10674

Hao Wang, Guozhi Wang, Han Xiao et al. · April 12, 2026
Agentic RL Self-Distillation Dense Supervision
核心贡献：解决多轮 LLM 智能体 RL 训练在稀疏奖励与长视野下样本效率低的问题。现有 On-Policy Self-Distillation (OPSD) 依赖能看到 ground-truth 的 privileged teacher 提供密集 token-level 监督，但固定的 privileged info 无法覆盖 agent 任务中多样的有效策略，与 RL 直接组合常常引发训练坍缩。

方法：提出 Skill-SD，将智能体自身已完成的轨迹总结为紧凑的自然语言 "skill"（描述成功行为、错误与工作流），作为动态 privileged info 仅提供给 teacher。学生始终在普通 task prompt 下行动，通过 distillation 学会内化这些指导。skill 来自 agent 本身，因此多样且自适应，不会强行灌输单一策略。

Key Takeaways

  • 把轨迹总结成自然语言 skill，是把 RL 稀疏信号转化为 dense 监督的新思路
  • teacher/student 的不对称：teacher 看到 skill，student 只看任务——避免了推断时依赖
  • 提供了 OPSD + RL 组合不坍缩的可操作方案

AgentGL: Towards Agentic Graph Learning with LLMs via Reinforcement Learning2604.05846

Yuanfu Sun, Kang Li, Dongzhe Fan, Jiajin Liu, Qiaoyu Tan · April 7, 2026
Agentic RL Graph Learning Curriculum RL
核心贡献：首个面向 Agentic Graph Learning (AGL) 的 RL 框架。现有智能体框架把外部信息当成非结构化文本，忽略了真实数据中的拓扑依赖。AgentGL 将图学习重新表述为"拓扑感知导航 + LLM 推理"交替的过程。

方法：(1) 给 LLM 智能体装配 graph-native 工具支持多尺度结构探索；(2) 通过 search-constrained thinking 偏向于额外图查询之前先做反思；(3) graph-conditioned curriculum RL 策略——渐进式加大拓扑探索难度并整合多面向奖励，用来稳定没有 step-wise 监督下的长视野策略学习。

Key Takeaways

  • 图结构是 agentic tool use 的新战场——拓扑依赖被显式暴露给 agent
  • graph-conditioned curriculum 是稳定长视野 agentic RL 的关键技巧

COVERT: Controllable and Verifiable Tool-Use Data Synthesis for Agentic RL2604.09813

Penn State, Amazon, Georgia Tech · April 10, 2026
Infrastructure Tool Use Data Synthesis
核心贡献：把现有 tool-use 语料从"离线 SFT 友好"升级为"RL 友好"——为可执行环境提供 reward-checkable online rollouts。

方法：两阶段 pipeline。Stage 1：通过 self-evolving 合成 + multi-level validation 生成可靠的基础工具调用轨迹。Stage 2：施加 oracle-preserving 增广——引入干扰工具、模糊/间接的用户查询、噪声/多格式/错误的工具输出，但严格保留 oracle 工具调用和最终答案作为 ground truth。由此通过引用匹配实现自动 reward，在特殊行为（如错误检测）上用轻量 judge 验证。

Key Takeaways

  • "Oracle-preserving augmentation" 提供了一个通用的 RL 数据增广范式
  • noise/distractor tool 是评估 agentic rollout 鲁棒性的关键设计

AnomalyAgent: Agentic Industrial Anomaly Synthesis via Tool-Augmented RL2604.07900

April 9, 2026
Agentic RL Tool Use Industrial
核心贡献：将 agentic RL 带入工业异常数据合成。配备 5 类工具（PG / IG / QE / KR / MG）实现闭环优化，SFT + RL 两阶段训练。奖励分三部分：任务奖励（异常的质量与位置合理性）、反思奖励（合成 prompt 改进能力）、行为奖励（轨迹一致性）。

在 MVTec-AD 数据集上，AnomalyAgent 达到 IS/IC-L = 2.10/0.33；ResNet34 分类精度 57.0%；UNet 在图像/像素级 AP 分别达到 99.3% / 74.2%，全面超越 zero-shot SOTA。

Key Takeaways

  • "行为奖励 + 反思奖励 + 任务奖励"三元组是 agentic 合成的通用奖励结构
  • RL 能让工业 anomaly synthesis 从一次性 prompt 升级为可迭代的闭环

II. Process Reward & Reward Modeling · 过程奖励与奖励建模

PROGRS: LLM Reasoning with Process Rewards for Outcome-Guided Steps2604.02341

April 3, 2026
Process Reward GRPO Reward Hacking
核心贡献：解决 PRM 分数与最终正确性不完全对齐、易奖励"流畅但错误"推理的问题。PROGRS 把过程奖励当作 outcome 组内的相对偏好而非绝对目标来使用。

方法：(1) Outcome-conditioned centering：将错误轨迹的 PRM 分数在每个 prompt 组内零均值化，去除系统偏差但保留信息性排序；(2) 用 frozen quantile-regression PRM 结合 multi-scale coherence evaluator 惩罚短窗口内的置信度波动；(3) 将最终 centered process bonus 整合进 GRPO，无需辅助目标或额外可训练组件。

在 MATH-500 / AMC / AIME / MinervaMath / OlympiadBench 上，PROGRS 在 fewer rollouts 下一致优于仅用 outcome 的基线。

Key Takeaways

  • 把 PRM 当"组内相对偏好"而非"绝对值"用，是避免 reward hacking 的关键
  • centering within incorrect group 保留了有用的排序信号、去掉了 systematic bias
  • 无需额外可训练模块即可嵌入 GRPO——易于部署

Process Reward Agents for Steering Knowledge-Intensive Reasoning2604.09482

ETH Zürich, Heidelberg University · April 10, 2026
Process Reward Medical Test-time
核心贡献：把 PRM 从"训练时组件"扩展为"测试时可插拔推理引导器"。知识密集型域（如医学）中，中间步骤通常不可本地验证，因为要跨大型外部知识源综合线索。细微错误会在推理链中传播却不被察觉。

方法：Process Reward Agents (PRA)：在测试时为 frozen policy 提供领域接地的、在线的 step-wise 奖励。与以前的 retrieval-augmented PRM 不同，PRA 允许基于搜索的 decoding 在每一生成步对候选轨迹排序并剪枝。

结果：Qwen3-4B + PRA 在 MedQA 上达到 80.8% 精度（4B 规模新 SOTA）。对 0.5B–8B 的未见 frozen policy 也能提升最多 25.7%，且无需更新 policy。

Key Takeaways

  • PRM 可以完全解耦于 policy，成为"可热插拔"的领域模块
  • 在知识密集型推理上，测试时搜索 + PRA 比训练更大模型更有性价比

Rubrics to Tokens: Bridging Response-level Rubrics and Token-level Rewards2604.02795

April 3, 2026 · GitHub
Reward Modeling Rubric Credit Assignment
核心贡献：针对 rubric-based RL 长期存在的"response-level 奖励稀疏且模糊"问题，给出 token 级归因方案。

方法：RTT 引入一个 Token-Level Relevance Discriminator，用以预测哪些 token 与某条约束相关。然后通过 RTT-GRPO 在统一框架中整合 response 级与 token 级 advantage，实现细粒度信用分配。在多个模型与规模上，RTT 在 instruction-level 和 rubric-level 精度上一致优于所有基线。

Key Takeaways

  • rubric-based RL 的下一步是 token 级信用——而非仅 response 级
  • Relevance Discriminator 可作为通用的"约束归因"模块插入其他 RL 流程

SubSearch: Intermediate Rewards for Unsupervised Guided Reasoning in Complex Retrieval2604.07415

Roxana Petcu, Evangelos Kanoulas, Maarten de Rijke · IRLab, University of Amsterdam · April 8, 2026
Process Reward Retrieval Unsupervised
核心贡献：针对复杂检索中多步推理无明确路径、只能得到稀疏 outcome 信号的问题，提出直接用内在过程奖励优化生成器，不依赖外部过程监督。

方法：SubSearch 将 outcome-only 监督切换为 intermediate reward，激励模型规划高质量的推理轨迹。与先前的 PRM 工作不同，奖励来自模型/系统内部派生的信号，而非外部标注。

Key Takeaways

  • "内在过程奖励"思路让 PRM 摆脱标注瓶颈——把奖励从监督学习变成系统属性
  • 检索系统成为 agentic RL 的一大催生场景

III. Tool Use & Multi-Hop Reasoning · 工具使用与多跳推理

TInR: Exploring Tool-Internalized Reasoning in Large Language Models2604.10788

April 12, 2026
Tool Use Knowledge Internalization Three-stage Training
核心贡献：提出"工具内化"新范式。现有 Tool-Integrated Reasoning (TIR) 在推理时依赖外部工具文档，导致工具掌握困难、工具规模受限、推理效率低。TInR 的主张是：把工具知识内化到 LLM 权重中，让模型在无需文档的情况下高效调用工具。

  • 工具文档从"推理时检索"转为"训练时内化"，是突破工具规模瓶颈的新思路

  • CAD 是现实世界中 agentic RL 的"终极长视野"测试场

KG-Reasoner: Reinforced End-to-End Multi-Hop Knowledge Graph Reasoning2604.12487

Shuai Wang, Yinan Yu · April 14, 2026
Multi-hop KG Reasoning End-to-End
核心贡献：解决 LLM 在 KG 多跳推理中因 pipeline 碎片化、步骤相互孤立而效能下降的问题。

方法：KG-Reasoner 将多步推理整合进 Reasoning LLM 单一"思考"阶段；通过 RL 让模型内化 KG 遍历过程，动态探索推理路径，并在必要时回溯。在 8 个多跳 / 知识密集型推理 benchmark 上达到或超过 SOTA。

  • 将"图遍历"放进"思考阶段"——而非流程上游——是多跳推理的结构性改进
  • 回溯 (backtracking) 能力是 RL 训练出的隐含能力

TRACE: Experiential Framework for Coherent Multi-hop KGQA2604.11193

Yingxu Wang, Jiaxin Huang, Mengzhu Wang, Nan Yin · April 13, 2026
Multi-hop KGQA Experience Replay
核心贡献：现有 KGQA 方法把每个推理步视为独立事件，难以利用先前探索的经验，导致碎片化推理和冗余探索。TRACE 统一 LLM 上下文推理与探索先验整合，增强多跳 KGQA 的一致性与鲁棒性。

方法：(1) 动态把不断演化的推理路径翻译成自然语言叙事以保持语义连贯；(2) 把先前探索轨迹抽象成可复用的"经验先验"，捕捉反复出现的探索模式；(3) dual-feedback re-ranking 机制在关系选择阶段融合叙事与先验。

Key Takeaways

  • 将探索历史作为"学到的先验"而非一次性上下文，是 RL 多跳推理的重要可复用资产
  • 自然语言叙事 + 经验先验的组合，为 reasoning trace 提供了"语义 + 结构"双重锚点

IV. Reasoning Efficiency & Calibration · 推理效率与校准

Graph-Based Chain-of-Thought Pruning for Reducing Redundant Reflections2604.05643

Hongyuan Yuan et al. · April 7, 2026
Reasoning RL Efficiency DPO + GRPO
核心贡献：系统识别 RL-trained reasoning LLM 中的两类冗余：不加选择的反思 (Indiscriminate Reflection) 和重复反思 (Repetitive Reflection)。

方法：将每条线性 CoT 转为 DAG，用双层剪枝策略：(1) 分支级剪枝去掉贡献弱的反思分支，(2) 深度级剪枝去除后期再验证。随后三阶段蒸馏：① SFT 用剪枝后的简洁轨迹初始化策略；② DPO 偏好"正确且更简洁"的轨迹；③ GRPO with length penalty 联合优化正确性与效率。

结果：平均推理 token 降低 42%，精度保持或提升。

Key Takeaways

  • "反思 ≠ 深思"——不是所有反思都有价值
  • SFT → DPO → GRPO 三段式 pipeline 是"效率 + 精度"联合优化的合理分工

CURE: Think Through Uncertainty — Long-Form Factuality via Reasoning Calibration2604.12046

Xin Liu, Lu Wang · University of Michigan · April 13, 2026
Factuality Calibration Claim-level
核心贡献：现有长文事实性方法或依赖 post-hoc 改写，或用 correctness-based RL 奖励，但都未教会模型估计自身生成的各部分可信度。现有方法输出一个整体标量置信度，对长文不足——不同 claim 的不确定性差异巨大。

方法：CURE 通过Claim-Aware Reasoning Protocol 把输出结构化为原子 claim + 显式置信估计；再用多阶段训练流程：先对齐置信度与正确性，再对事实性做 RL 优化。

结果：在 4 个长文事实性 benchmark 上一致优于监督与 RL 基线；Biography 上 claim 级精度提升最多 39.9%，且不牺牲召回率。

Key Takeaways

  • "整体置信度"粒度不够——claim-level calibration 是长文事实性的下一个关键
  • 先对齐置信度再优化事实性——这种 curriculum 让 RL 信号更稳定

V. RL at Domain Boundaries · RL 在现实域边界的探索

When Valid Signals Fail: Regime Boundaries Between LLM Features and RL Trading Policies2604.10996

Zhengzhe Yang · April 13, 2026
RL Finance Distribution Shift
核心贡献：提出一个警示性的负面结果：LLM 提取的"客观有效"信号不一定能转化为下游 RL 任务的性能提升。作者构建模块化 pipeline：frozen LLM 作为无状态特征提取器，把新闻与财报转换为固定维向量，喂给下游 PPO 智能体。

VII. Cross-Cutting Themes · 交叉主题与研究脉络

本期综合观察 · Synthesis

本期 19 篇新论文呈现出五条清晰的研究脉络：

1. 从"奖励是什么"到"奖励怎么用"——PROGRS / RTT / SubSearch / E-GRM / PRA 几乎同时出现，说明社区已不再纠结于"reward model 架构"，而是聚焦如何将 reward 分配到 token / step / claim 级。这与晨报中 SKPO/TEPO/SPPO 的 credit-assignment 主题呼应，形成"细粒度信用分配"的合奏。

2. Agentic RL 的"反思阶段"——RAGEN-2 的模板坍缩诊断是一个里程碑：社区开始意识到稳定 entropy / loss curve 并不等于稳定的推理质量。CoT Pruning 对"反思的反思"、TRACE 对"经验先验"的引入，都指向一个共同方向：Agentic RL 需要跨 rollout 的信息流，而不仅是单轨迹内的奖励。

3. Tool Use 从"显式"走向"内化"——TInR 主张把工具知识固化到权重中，代表一种可能的终局：当工具库足够标准化，工具的边界会模糊化，最终变为模型的隐式能力。这与晨报中的 TPO 统一视角有哲学上的呼应。

4. RLVR 的 domain-aware 扩张——Regime Boundaries、OOM-RL、Argumentation Editing 三篇论文一起描绘了 RLVR 在非稳态/真实世界中的边界：verifiable reward 在规则明确的领域（代码、数学、Sharpe ratio）极为强大，但在 regime shift 下会失效。OOM-RL 的 "capital depletion as negative gradient" 是一种极端但深刻的应对。

5. 数据基础设施成为一等公民——COVERT、AnomalyAgent、Skill-SD 的共同点是：不再简单从静态数据集训练，而是构造合成的、可验证的、能 rollout 的交互环境。这表明 Agentic RL 已经过了"证明可行"的阶段，进入"工程化规模化"阶段。
